## 1. 系统简介

### 1.1 项目定位

**地球村民监察互助联盟**是一个网友自发组建的就业歧视与剥削行为监督平台。通过实名投票、证据上传机制，形成透明化榜单，推动社会对企业用工环境的关注与改善。

### 1.2 四个观察榜

| 榜单分类 | 代码 | 说明 |
|---------|------|------|
| 35 岁以上就业歧视观察榜 | `age_discrimination` | 记录存在年龄歧视行为的企业 |
| 性别歧视观察榜 | `gender_discrimination` | 记录存在性别歧视行为的企业 |
| 职场 PUA 专制观察榜 | `pua_despotism` | 记录存在职场精神控制、专制管理的企业 |
| 过劳剥削观察榜 | `overwork_exploitation` | 记录存在强制加班、无加班费的企业 |

### 1.3 核心原则

- **匿名投票**: 所有投票者信息严格保密，只公开统计结果
- **证据优先**: 鼓励上传截图/文档作为佐证材料
- **防刷票机制**: 单用户每日最多投 3 票，同一企业对同类别不可重复投票
- **真实可靠**: 通过设备指纹、IP 频率限制等技术手段保障数据真实性

## 2. 快速开始

### 2.1 Web 端操作流程

#### 步骤 1: 注册账号

1. 访问系统（本地：`http://127.0.0.1:8000/`）
2. 点击「注册」按钮
3. 输入 11 位手机号
4. 获取验证码（测试环境下，控制台会输出验证码）
5. 设置密码（至少 8 位，包含字母和数字）
6. 提交完成注册

**注意**: 
- 新注册用户需等待管理员审核
- 同一手机号不可重复注册

#### 步骤 2: 登录系统

1. 在登录页面输入手机号和密码
2. 点击「登录」
3. 首次登录成功后获得 Token，用于后续 API 调用（Web 端自动处理）

#### 步骤 3: 提交投票

1. 登录后进入「投票」页面
2. 选择目标榜单类别
3. 输入企业名称进行搜索并选中
4. 填写投票理由（最多 2000 字）
5. 上传证据材料（可选，推荐上传）
   - 支持格式：图片（JPG/PNG）、文档（PDF/DOCX）
   - 文件大小限制：不超过 5MB
6. 确认提交

**投票后**:
- 当日剩余票数会在提交成功后显示
- 已投票的企业在同类别下不可重复投票
- 可上传证据，但每个投票仅限一次

#### 步骤 4: 查看榜单排行

1. 登录后访问「榜单」页面
2. 可选择：
   - **全部榜单概览**: 查看各分类的 Top N 企业
   - **特定榜单详情**: 点击具体榜单名称查看详情

**榜单展示内容**:
- 排名（根据票数从高到低）
- 企业名称、行业、所在城市
- 得票数和得票率（%）

### 3.2 投票流程详解

#### 企业搜索 API

```bash
GET /api/vote/companies/search/?q=华为

# 参数说明
- q: 搜索关键词（支持模糊匹配，如"腾讯"可找到"Tencent Tech"）
- 返回最多 50 条结果
```

**搜索结果格式**:
```json
[
    {
        "id": 11,
        "name": "华为技术",
        "industry": "通信设备",
        "city": "深圳"
    }
]
```

#### 提交投票详细步骤

1. **准备阶段**
   - 确认目标企业已存在于系统（若不存在可联系管理员添加）
   - 收集证据材料（推荐：聊天记录截图、加班通知邮件、工资条等）

2. **填写表单**
   ```json
   POST /api/vote/submit/
   
   {
       "category": "age_discrimination",      # 投票分类（必填）
       "company_id": 13,                       # 企业 ID（从搜索结果获取，必填）
       "reason": "该企业招聘 JD 明确要求 35 岁以上拒收，构成年龄歧视",  # 理由描述（可选，建议填写）
   }
   ```

3. **提交后反馈**
   - 成功：返回 `vote_id` 和剩余票数
   - 失败：可能原因包括
     - 今日投票已达上限（429 Too Many Requests）
     - 该企业已投过同类别票（400 Bad Request）
     - 企业不存在/分类无效

### 3.3 证据上传详解

#### 支持的文件格式

| 类型 | 扩展名 | MIME 类型 | 说明 |
|------|--------|----------|------|
| 图片 | .jpg, .jpeg, .png | image/jpeg / image/png | 推荐截图类证据（聊天记录、通知等） |
| 文档 | .pdf | application/pdf | PDF 扫描件或导出的邮件 |
| 文档 | .docx | application/vnd.openxmlformats-officedocument.wordprocessingml.document | Word 文档 |

#### 上传流程

1. **获取投票 ID**
   - 提交投票成功后立即返回 `vote_id`
   - 或通过「我的投票」页面查看历史投票记录（未来功能）

2. **调用上传 API**
   ```python
   import requests
   
   s = requests.Session()
   s.headers['Authorization'] = f'Token {your_token}'
   
   evidence_resp = s.post(
       'http://127.0.0.1:8000/api/vote/evidence/upload/',
       data={'vote_id': 5},                      # 对应投票的 ID
       files={
           'evidence_file': (                   # 文件名（必填字段名）
               'proof.pdf',                     # 自定义文件名
               open('evidence_content.pdf', 'rb'),  # 文件内容
               'application/pdf'                # MIME 类型
           )
       }
   )
   ```

3. **结果验证**
   - 成功：返回 `file_url` 和上传时间
   - 失败原因：
     - 该投票已有证据（不可重复提交）
     - 文件格式不支持
     - 文件过大（>5MB）

#### 证据管理建议

- **命名规范**: 建议使用描述性文件名，如"2024-03-15_加班通知.png"
- **内容要求**: 确保截图/文档包含关键信息（时间、企业名称、事件经过）
- **隐私保护**: 上传前请对个人敏感信息打码处理

```bash
    "category": "age_discrimination",
    "category_name": "35 岁以上就业歧视观察榜",

        {
            "rank": 1,                           # 排名（从 1 开始）
```

#### 榜单排序规则

- 按得票数 `total_votes` **从高到低**排序
- 若票数相同，按得票率 `vote_rate` **从高到低**排序
- 每类榜目前显示前 20 名（未来可扩展）

## 4. 常见问题解答

### Q1: 今日投票次数已达上限怎么办？

**A**: 
- 规则：同一用户/日最大可投 3 票（不分类别）
- 解决方案：明日零点后可继续使用，或注册新账号（建议遵守公平原则）

**技术细节**:
```python
# IP + 设备指纹双重验证防止刷票
daily_votes = Vote.objects.filter(
    voter_id=user_id,
    category=category,
    ip_address__startswith=request_ip,
    created_at__gte=today_midnight
).count()
```

### Q2: 已对企业投过同类别票，能否修改或删除？

**A**: 
- 当前版本不支持修改或删除已有投票
- 如确需更正，请联系管理员（`11011912000 / 110119120`）处理

### Q3: 证据上传失败怎么办？

**A**: 
检查以下条件：
1. **文件格式**: 仅支持 `.jpg`, `.jpeg`, `.png`, `.pdf`, `.docx`
2. **文件大小**: 不超过 5MB（建议<2MB 以提高兼容性）
3. **重复上传**: 每票仅限一次证据提交

**调试命令**:
```bash
# 检查文件类型
file proof.pdf

# 查看文件大小
ls -lh proof.pdf
```

### Q4: 如何确认投票已成功提交？

**A**: 
成功返回示例：
```json
{
    "message": "投票成功",
    "vote_id": 6,
    "remaining_today": 0          # 今日剩余票数
}
```

若返回错误码 `201`，请检查:
- 企业是否存在（尝试重新搜索确认）
- 分类是否有效（见第 3.2 节投票流程详解）

### Q5: 数据安全性如何保障？

**A**: 
- **手机号加密存储**: Fernet 对称加密算法，私钥从环境变量读取
- **Token 认证**: 每次 API 调用需携带 Token，有效期 24 小时
- **防刷票机制**: IP+ 设备指纹双重验证
- **证据材料安全**: 文件独立存储于 `media/evidence/` 目录

### Q6: 作为管理员如何管理后台数据？

**A**: 
1. 访问 `/admin/`（需预置管理员账号）
2. 查看/编辑企业列表
3. 审核用户提交的数据
4. 统计榜单排名更新情况

## 5. API 使用示例

### 5.1 Python Requests 完整测试脚本

```python
import requests
from rest_framework.authtoken.models import Token
import os

# 环境变量配置
BASE_URL = 'http://127.0.0.1:8000'

def main():
    # 登录获取 Token
    login_resp = requests.post(
        f'{BASE_URL}/api/auth/login/',
        json={'username': 'testuser', 'password': 'TestPass123!'}
    )
    
    if login_resp.status_code != 200:
        print(f'登录失败：{login_resp.text}')
        return
    
    token = login_resp.json()['token']
    s = requests.Session()
    s.headers['Authorization'] = f'Token {token}'
    
    # 1. 搜索企业
    comp_resp = s.get(
        f'{BASE_URL}/api/vote/companies/search/',
        params={'q': '华为'}
    )
    companies = comp_resp.json()
    print(f'找到企业：{[c["name"] for c in companies]}')
    
    # 2. 提交投票
    vote_resp = s.post(
        f'{BASE_URL}/api/vote/submit/',
        data={
            'category': 'age_discrimination',
            'company_id': companies[0]['id'],
            'reason': '该企业存在年龄歧视行为'
        }
    )
    
    if vote_resp.status_code == 201:
        print(f'投票成功！Vote ID={vote_resp.json()["vote_id"]}')
        
        # 3. 上传证据（如果投票成功）
        evidence_resp = s.post(
            f'{BASE_URL}/api/vote/evidence/upload/',
            data={'vote_id': vote_resp.json()['vote_id']},
            files={
                'evidence_file': (
                    'proof.pdf',
                    b'%PDF-1.4\n证据内容',
                    'application/pdf'
                )
            }
        )
        
        if evidence_resp.status_code == 201:
            print(f'证据上传成功：{evidence_resp.json()["file_url"]}')
    else:
        print(f'投票失败：{vote_resp.text}')

if __name__ == '__main__':
    main()
```

### 5.3 JavaScript (Fetch API) 示例

```javascript
const BASE_URL = 'http://127.0.0.1:8000';

// 登录获取 Token
async function login() {
    const resp = await fetch(`${BASE_URL}/api/auth/login/`, {
        method: 'POST',
        headers: {'Content-Type': 'application/json'},
        body: JSON.stringify({username: 'testuser', password: 'TestPass123!'})
    });
    
    if (!resp.ok) throw new Error('登录失败');
    const data = await resp.json();
    return data.token;
}

// 搜索企业
async function searchCompanies(keyword, token) {
    const resp = await fetch(
        `${BASE_URL}/api/vote/companies/search/?q=${keyword}`,
        {headers: {'Authorization': `Token ${token}`}}
    );
    return resp.json();
}

// 提交投票
async function submitVote(token, category, companyId, reason) {
    const resp = await fetch(`${BASE_URL}/api/vote/submit/`, {
        method: 'POST',
        headers: {'Authorization': `Token ${token}`, 'Content-Type': 'application/json'},
        body: JSON.stringify({category, company_id: companyId, reason})
    });
    return resp.json();
}

        method: 'POST',

}
```

## 附录：错误代码速查表

| HTTP 状态码 | 说明 | 可能原因 | 解决方案 |
|------------|------|----------|----------|
| 200 OK | 成功 | API 正常返回 | 无操作 |
| 201 Created | 资源创建成功 | 投票/证据上传完成 | 保存返回数据 |
| 400 Bad Request | 请求错误 | 参数缺失、格式无效、重复提交 | 检查输入参数 |
| 401 Unauthorized | 未认证 | Token 缺失或无效 | 重新登录获取新 Token |
| 403 Forbidden | 权限拒绝 | 非投票人尝试上传他人证据 | 使用创建者账号 |
| 429 Too Many Requests | 频率超限 | 单用户/日超 3 票 | 次日继续使用 |
